### The `data/` Directory detail
This directory stores the dataset at various stages of the pipeline, ensuring full reproducibility of the preprocessing and splitting logic.

* `cirrhosis.csv`
  The **raw dataset** sourced from the Mayo Clinic study (via Kaggle). It contains the original 418 patient records with all initial clinical variables and missing values.

* `dataset_refined.csv`
  The **clean, preprocessed dataset**. This version represents the data after the cleaning pipeline:
  * **Data Cleaning**: Samples with Target missing values, and with more than 2 features missing values are removed.
  * **Imputation:** Missing values filled via KNN and Mode imputation.
  * **Leakage Removal:** Exclusion of `N_Days` and `Status`.
  * **Scaling:** Application of Z-score normalization to numerical features.

* `data_refined_stratifikfold.csv`
  The **stratified operational dataset**. It includes an additional metadata column that assigns each patient to a specific partition:
  * **Folds 1-5:** Assigned to the Development Set for Cross-Validation.
  * **Benchmark:** Assigned to the held-out Test Set.
  This file is crucial to guarantee that the exact random splits used in the report are preserved for anyone running the code.

* `best_features_list_*.pkl`
  Serialized lists containing the **optimal feature subsets** (the 11 biomarkers) identified after the **Stability Selection** protocol for each architecture (`lr`, `svm`, `tree`).

* `benchmark_prediction_*.pkl`
  Stores the final **inference results** on the independent Benchmark Set. These files contain the predictions generated by the optimized models, allowing for the reproduction of the final confusion matrices and metrics without retraining.
